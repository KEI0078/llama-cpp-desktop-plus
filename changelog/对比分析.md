| 功能 | 位置 |
|------|------|
| 参数 11 分类面板 | ✅ 概述→运行参数 |
| GPU 实时解析 | ✅ 概述→已检测的显卡 |
| MTP 模型支持 | ✅ 模型→模型与模板 |
| 文件路径选择 | ✅ Qiao 原生 pickFile |
| llama-server 启动管理 | ✅ Qiao 原生 |
| 聊天/对话测试 | ✅ Qiao 原生 |
| 参数实时保存 | ✅ system.json |
| llama.cpp Web UI 入口 | ✅ 展示开关 |
| API 配置 | ✅ Qiao 原生 |
| 上下文批量测速 | ✅ 日志→快速测速 |
| CPU 多线程测试 | ✅ 日志→快速测速 |
| llama-bench 基准 | ✅ 日志→快速测速 |
| 解析启动参数 | ✅ 开发者→解析参数按钮 |
| 保存启动脚本 | ✅ 开发者→保存为脚本 |
| 加载启动脚本 | ✅ 开发者→加载脚本 |
| 模型配置保存/加载 | ✅ 开发者→模型配置 |
| 实时停止测速 | ✅ 红色停止按钮 |

## ❌ 还未做的

| 功能 | 说明 |
|------|------|
| **并发压力测试** | 多请求并发压测服务端稳定性。cn.LammaForms 有此功能，需另开线程发多个 HTTP 请求 |
| **模型自动扫描** | btn_ggufModel_Refresh，从目录自动检测 .gguf 文件列表。Qiao 手动选路径 |
| **日志三栏切换** | 运行/测试/服务端三栏分开。现在合在一个页面 |
| **electron-builder 打包** | 生成安装程序 exe。`npm run dist` 需要先配置 |

## 📊 总结

核心功能基本都覆盖了。剩下的 4 个中：
- **并发压力测试** — 最值得补，对调试服务稳定性有用
- **模型自动扫描** — 方便但非必须
- **日志三栏** — 用户已选择合并到一个页面
- **打包** — 需要时再配置
